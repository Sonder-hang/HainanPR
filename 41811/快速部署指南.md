# 快速部署指南

本文档面向**初次拿到项目**的开发者，按顺序执行即可运行。

---

## 第一步：环境准备

| 工具 | 版本 | 下载地址 |
|------|------|---------|
| Python | 3.9+ | https://www.python.org/downloads/ |
| Node.js | 18+ | https://nodejs.org/ |
| MySQL | 8.0+ | https://dev.mysql.com/downloads/mysql/ |

> 安装时勾选"Add Python to PATH"。MySQL 安装过程中会提示设置 root 密码，记住它，后面要用。

---

## 第二步：创建数据库

打开 MySQL 命令行（Windows 可用 MySQL Workbench 或 `cmd`），执行：

```sql
CREATE DATABASE hainan_41811 CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

---

## 第三步：修改密码配置

项目脚本中的 MySQL 密码默认是 `123456`，如果你的密码不同，改以下两处：

**1. `rebuild_tables.py` 第 3 行：**
```python
DEST = dict(host="127.0.0.1", user="root", password="你的密码", database="hainan_41811", charset="utf8mb4")
```

**2. `backend_fastapi/.env` 中的 DATABASE_URL：**
```
DATABASE_URL=mysql+pymysql://root:你的密码@127.0.0.1:3306/hainan_41811
```

---

## 第四步：安装后端依赖并启动

```bash
cd backend_fastapi
pip install -r requirements.txt
python main.py
```

看到类似以下输出说明启动成功：
```
Uvicorn running on http://0.0.0.0:8000
```

> 后端地址：http://localhost:8000
> API 文档：http://localhost:8000/docs

---

## 第五步：初始化数据库表

**重新打开一个终端窗口**，执行：

```bash
python rebuild_tables.py
```

看到 `表重建完成！` 即成功。

> 如果 `backend_fastapi/db.sqlite3` 存在旧数据，还需要执行迁移：
> ```bash
> python migrate_to_mysql.py
> ```

---

## 第六步：安装前端依赖并启动

**再打开一个终端窗口**，执行：

```bash
cd my-vue-app
npm install
npm run dev
```

看到类似以下输出说明启动成功：
```
VITE v5.x.x  ready in xxx ms
Local: http://localhost:5173/
```

> 前端地址：http://localhost:5173

---

## 第七步：启动 Text2SQL 服务（可选）

如果需要"自然语言转 SQL"功能，还需要启动 Text2SQL 服务。

**再打开一个终端窗口**，执行：

```bash
cd backend_fastapi/Hainan_SQL-main/text2sql_app
pip install -r requirements.txt
python run.py
```

看到类似以下输出说明启动成功：
```
Uvicorn running on http://0.0.0.0:8001
```

> Text2SQL 地址：http://localhost:8001

---

## 一键启动脚本（Windows）

项目根目录提供了简化脚本，执行前先改好密码：

```powershell
# 启动所有服务
.\start_local.ps1

# 停止所有服务
.\stop_local.ps1
```

---

## 常见问题

**Q: 启动后端报错 `ModuleNotFoundError`**
```bash
cd backend_fastapi
.venv\Scripts\activate          # Windows
pip install -r requirements.txt
python main.py
```

**Q: SQL 执行报错语法错误**
确保 `rebuild_tables.py` 密码和数据库名与 MySQL 配置一致，然后重新运行。

**Q: 前端页面打不开**
确认后端在 8000 端口正常运行，前端 `.env` 中 `VITE_API_BASE_URL=http://localhost:8000`。

**Q: 修改了后端模型后表结构没更新**
```bash
python rebuild_tables.py          # 重建表（会清空数据）
python migrate_to_mysql.py        # 恢复数据（如果有）
```
