# 快速部署指南

本文档面向**初次拿到项目**的开发者，按顺序执行即可运行。

---

## 环境准备

| 工具 | 版本 | 下载地址 |
|------|------|---------|
| Python | 3.9+ | https://www.python.org/downloads/ |
| Node.js | 18+ | https://nodejs.org/ |
| MySQL | 8.0+ | https://dev.mysql.com/downloads/mysql/ |
| Redis | 6.0+ | https://redis.io/docs/getting-started/ |

> 安装 Python 时请勾选"Add Python to PATH"。MySQL 安装过程中会提示设置 root 密码，记住它，后面要用。

---

## 第一步：启动 Redis

Redis 是指标异步执行的必须组件。

```bash
# macOS
brew install redis
brew services start redis

# Ubuntu/Debian
sudo apt install redis-server
sudo systemctl start redis

# 验证
redis-cli ping
# 应返回 PONG
```

---

## 第二步：创建数据库

打开 MySQL 命令行，执行：

```sql
CREATE DATABASE hainan_41811 CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

---

## 第三步：导入数据库初始化 SQL

```bash
mysql -u root -p hainan_41811 < hainan-18core-latest.sql
```

看到无报错说明导入成功。

> 此 SQL 包含所有表结构和十八项核心制度指标的初始配置。

---

## 第四步：修改密码配置

项目中的 MySQL 密码默认是 `123456`，如果你的密码不同，改以下两处：

**1. `backend_fastapi/.env` 中的 DATABASE_URL：**
```
DATABASE_URL=mysql+pymysql://root:你的密码@127.0.0.1:3306/hainan_41811
```

**2. `backend_fastapi/.env` 中的 CELERY 和 REDIS 配置（按需修改）：**
```
CELERY_BROKER_URL=redis://127.0.0.1:6379/0
CELERY_RESULT_BACKEND=redis://127.0.0.1:6379/0
REDIS_URL=redis://127.0.0.1:6379/0
```

---

## 第五步：安装后端依赖并启动

```bash
cd backend_fastapi
pip install -r requirements.txt
python -m uvicorn main:app --host 0.0.0.0 --port 8001
```

看到以下输出说明启动成功：
```
INFO:     Uvicorn running on http://0.0.0.0:8001
```

> 后端地址：http://localhost:8001
> API 文档：http://localhost:8001/docs

---

## 第六步：启动 Celery Worker（必须）

**再打开一个终端窗口**，执行：

```bash
cd backend_fastapi
python3 -m celery -A app.celery_app worker --loglevel=info
```

看到以下输出说明启动成功：
```
[INFO] celery@xxx ready

> Worker 启动后会自动连接到 Redis，指标执行将变为异步模式。
> 如果不启动 Worker，指标执行仍以同步模式运行（页面会等待执行完成）。
```

---

## 第七步：安装前端依赖并启动

**再打开一个终端窗口**，执行：

```bash
cd my-vue-app
npm install
npm run dev
```

看到以下输出说明启动成功：
```
VITE v8.x.x  ready in xxx ms
  ➜  Local:   http://localhost:5173/
```

> 前端地址：http://localhost:5173

---

## 一键启动脚本

项目根目录提供了 `start.sh` 和 `stop.sh`，可一键启动/停止所有服务（使用前请确保密码配置正确）：

```bash
# 启动所有服务
./start.sh

# 停止所有服务
./stop.sh
```

## 常见问题

**Q: 启动后端报错 `ModuleNotFoundError`**
```bash
cd backend_fastapi
pip install -r requirements.txt
python -m uvicorn main:app --port 8001
```

**Q: Redis 连接失败**
```bash
redis-cli ping  # 应返回 PONG
# 如果没反应，重启 Redis
brew services restart redis  # macOS
sudo systemctl restart redis  # Linux
```

**Q: 指标执行页面没有反应**
确认 Celery Worker 已启动（见第六步）。

**Q: 排行榜没有数据**
- 确认指标已在"指标管理"页面执行过
- 排行榜依赖 `indicator_execution.subitem_data` 字段，明细级 SQL（无 GROUP BY 聚合）可能无法生成排行榜数据

**Q: SQL 执行报错语法错误**
确保 `backend_fastapi/.env` 中密码和数据库名与 MySQL 配置一致。

**Q: 前端页面打不开**
确认后端在 8001 端口正常运行。

**Q: 修改了后端模型或需要重置数据**
```bash
mysql -u root -p hainan_41811 < hainan-18core-latest.sql
```
