## 模型对话策略分析

### 一、对话划分

系统共涉及 **3个对话**：

| 对话 | 用途 | 触发时机 |
|------|------|---------|
| **对话1** | 任务理解与前置检查 | 用户点击"开始分析" |
| **对话2** | SQL生成与验证 | 用户确认理解后 |
| **对话3** | 逻辑一致性检查 | 仅当结果为空时 |

---

### 二、每一步骤归属

| 步骤 | 所属对话 | 说明 |
|------|---------|------|
| 模型思考与自我反思 | 对话1 | 1次调用 |
| 内容管道（标识符识别） | 独立调用 | 结构化输出，不占对话 |
| 结构管道（SQL语法推导） | 独立调用 | 结构化输出，不占对话 |
| SQL生成 | 对话2 | 1次调用，可循环 |
| SQL修正 | 对话2 | 循环中调用 |
| SQL转自然语言 | 对话3 | 仅结果为空时 |
| 逻辑对比 | 对话3 | 仅结果为空时 |

---

### 三、对话实现方式

项目已有 `chat_completion_multiturn` 函数，支持多轮对话：

```python
def chat_completion_multiturn(
    messages: list[dict[str, str]],  # 完整的消息列表
    temperature: Optional[float] = None,
) -> tuple[str, Optional[str]]:
```

**实现方式**：手动构建完整的 messages 列表，每次调用时传入。不需要history拼接。

---

### 四、对话管理代码

```python
class LLMConversation:
    """对话管理器"""

    def __init__(self):
        self.messages: list[dict] = []

    def add_system(self, content: str):
        self.messages.append({"role": "system", "content": content})

    def add_user(self, content: str):
        self.messages.append({"role": "user", "content": content})

    def add_assistant(self, content: str):
        self.messages.append({"role": "assistant", "content": content})

    async def send(self, user_message: str) -> str:
        self.add_user(user_message)
        response, _ = chat_completion_multiturn(self.messages)
        self.add_assistant(response)
        return response


class SQLGeneratorService:

    def __init__(self):
        self.conv1 = LLMConversation()  # 对话1
        self.conv2 = LLMConversation()  # 对话2

    def start_new_task(self):
        self.conv1 = LLMConversation()
        self.conv2 = LLMConversation()

    async def step1_understand(self, tables, question):
        """对话1: 任务理解"""
        self.conv1.add_system("你是医疗数据分析专家...")
        return await self.conv1.send(f"表: {tables}\n需求: {question}")

    async def step2_generate_sql(self, understanding):
        """对话2: SQL生成"""
        self.conv2.add_system("你是SQL生成专家...")
        return await self.conv2.send(f"理解结果: {understanding}")

    async def step2_correct_sql(self, error):
        """对话2: SQL修正（循环中，自动携带对话历史）"""
        return await self.conv2.send(f"修正错误: {error}")

    async def step3_logic_check(self, sql, intent):
        """对话3: 逻辑检查（新建独立对话）"""
        conv3 = LLMConversation()
        conv3.add_system("你是逻辑检查专家...")
        nl = await conv3.send(f"SQL: {sql}")
        result = await conv3.send(f"原需求: {intent}\nSQL理解: {nl}")
        return result
```

    ▼

                 │
                 ▼
        ┌────────────────┐

    ┌───────────────────┐

    └───────────────────┘
                │
                ▼
┌─────────────────────────────────────┐
│  对话2: SQL生成与验证               │
│  • SQL生成                         │
│  • 执行验证（程序）                 │
│  • 结果验证（程序）                 │

└───────────────┬────────────────────┘
                │
                ▼
    ┌───────────────────┐
    │ 结果非空          │→ 返回结果
    │ 结果为空          │→ 触发对话3
    └─────────┬─────────┘
              │
              ▼
┌─────────────────────────────────────┐
│  对话3: 逻辑一致性检查              │
│  • SQL转自然语言（1次）             │
│  • 逻辑对比（1次）                 │
│  • 如不一致→回到对话2重新生成     │
└─────────────────────────────────────┘
```

|------|---------|------|---------|
| 对话1 | 1次 | 否 | messages列表 |
| 对话2 | 1-3次 | 是（最多3次） | messages列表自动累积 |
| 对话3 | 2次 | 否 | 新建独立messages列表 |
